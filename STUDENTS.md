# Памятка студента: как работать с учебным проектом

На протяжении курса мы строим один проект. Состояние к каждому уроку
зафиксировано тегом `lesson-01`, `lesson-02`, … `lesson-15`.

## Один раз — скачать репозиторий

```bash
git clone https://github.com/<логин>/sqst-course-project.git
cd sqst-course-project
```

## Перед каждым уроком — переключиться на нужный тег

```bash
git fetch --tags         # забрать новые теги, если появились
git checkout lesson-02   # подставьте номер текущего урока
```

Всё. Файлы в папке станут такими, какими они должны быть к началу урока.

> Git скажет что-то про «detached HEAD» — это нормально. Это значит
> «вы сейчас смотрите на фиксированный снимок, а не на живую ветку».
> Для занятий и экспериментов этого достаточно.

## Хочу поэкспериментировать и сохранить свои правки

Сделайте свою ветку от тега:

```bash
git checkout lesson-02
git switch -c my-lesson-02     # своя ветка от этого состояния
# …правите файлы, коммитите как обычно…
git add .
git commit -m "мои эксперименты"
```

Ваши правки никому не мешают и живут в своей ветке.

## Вернуться к «чистому» состоянию урока

```bash
git checkout lesson-02
```

(если есть незакоммиченные правки — git предупредит; либо сохраните их
коммитом в своей ветке, либо отмените через `git restore .`)

## Сравнить, что изменилось между уроками

В браузере удобнее всего:

```
https://github.com/<логин>/sqst-course-project/compare/lesson-01...lesson-02
```

Покажет все файлы и строчки, которые добавились/поменялись.

## Шпаргалка одной картинкой

| Хочу…                                | Команда                                  |
| ------------------------------------ | ---------------------------------------- |
| Скачать проект                       | `git clone <URL>`                        |
| Увидеть все теги                     | `git tag -l`                             |
| Забрать новые теги с сервера         | `git fetch --tags`                       |
| Переключиться на урок N              | `git checkout lesson-0N`                 |
| Сделать свою ветку для экспериментов | `git switch -c my-work`                  |
| Вернуться к состоянию урока          | `git checkout lesson-0N`                 |
| Посмотреть историю                   | `git log --oneline --decorate`           |

Если что-то пошло не так — не паникуйте, ничего безвозвратно не ломается.
Спросите в чате курса, поможем.
